The Refined Role-Based ModelWe will calculate a normalized micro_score and macro_score for every player based on their role.Laners (TOP, ADC) - The DuelistsMicro (Laning): laningPhaseGoldExpAdvantage (binary), maxCsAdvantageOnLaneOpponent (numeric), soloKills (numeric).Macro (Map): damageDealtToObjectives and First Tower participation.Mid Lane - The PivotMicro (Laning): Same as Top/ADC.Macro (Map): takedownsBefore15 (captures roaming bot/top) + damageDealtToObjectives.Junglers - The Map DictatorsMicro (Early Skirmishes): We will use killsOnLanerEarlyJungleAsJungler (successful early ganks) and takedownsBefore15. The Machine Learning Strategy (How we use this)You mentioned that weighting should be different per role (e.g., Top lane micro matters more than Support micro).Here is the beauty of Machine Learning: We don't have to hardcode the weights. If we feed the model these three binary outcomes for a Top Lane matchup:match_won: Truemicro_won: Truemacro_won: FalseOver 100,000 games, the ML algorithm will mathematically "discover" that for Top Laners, the correlation between micro_won and match_won is massive (e.g., $r = 0.75$), while the correlation for Supports might be low (e.g., $r = 0.20$).By organizing the database to simply track "Did Champion A beat Champion B in Micro/Macro?", the ML model will automatically construct those role-specific weights during training. It will learn that picking a lane-dominant Top Laner is critical, but picking a map-dominant Support is equally critical.3. The SQL Schema UpdateWe only need to add two columns to our matchup_stats table to track this vast amount of intelligence.Run this in your Supabase SQL Editor:SQL-- Add our new ML Truth Labels
ALTER TABLE matchup_stats 
ADD COLUMN champ_a_micro_wins INTEGER DEFAULT 0,
ADD COLUMN champ_a_macro_wins INTEGER DEFAULT 0;

-- Update the UPSERT function to handle them
CREATE OR REPLACE FUNCTION update_matchup_holistic(
    c_a INT, c_b INT, pos TEXT, t TEXT, dur TEXT, 
    a_won_game BOOLEAN, a_won_micro BOOLEAN, a_won_macro BOOLEAN
) RETURNS void AS $$
BEGIN
  INSERT INTO matchup_stats (
      champ_a_id, champ_b_id, position, rank_tier, duration_bucket, 
      matches, champ_a_wins, champ_a_micro_wins, champ_a_macro_wins
  )
  VALUES (
      c_a, c_b, pos, t, dur, 1, 
      CASE WHEN a_won_game THEN 1 ELSE 0 END,
      CASE WHEN a_won_micro THEN 1 ELSE 0 END,
      CASE WHEN a_won_macro THEN 1 ELSE 0 END
  )
  ON CONFLICT (champ_a_id, champ_b_id, position, rank_tier, duration_bucket)
  DO UPDATE SET 
    matches = matchup_stats.matches + 1,
    champ_a_wins = matchup_stats.champ_a_wins + CASE WHEN a_won_game THEN 1 ELSE 0 END,
    champ_a_micro_wins = matchup_stats.champ_a_micro_wins + CASE WHEN a_won_micro THEN 1 ELSE 0 END,
    champ_a_macro_wins = matchup_stats.champ_a_macro_wins + CASE WHEN a_won_macro THEN 1 ELSE 0 END;
END;
$$ LANGUAGE plpgsql;
4. The Final worker.py Process LogicHere is the reworked logic. We extract all those granular challenges you found, compress them into a micro_score and macro_score, and then compare them directly.Pythondef process_match_data(match_data):
    info = match_data.get('info', {})
    if info.get('gameMode') != "CLASSIC": return
    
    participants = info.get('participants', [])
    if len(participants) != 10: return 
    
    raw_tier = participants[0].get('tier', 'UNKNOWN')
    tier = raw_tier.upper() if raw_tier else "UNKNOWN"
    
    duration = info.get('gameDuration', 0)
    if duration < 1500: dur_bucket = "EARLY_0_25"
    elif duration < 2100: dur_bucket = "MID_25_35"
    else: dur_bucket = "LATE_35_PLUS"

    lanes = {} 

    for p in participants:
        pos = p.get('teamPosition')
        if not pos or pos == "": continue
        if pos not in lanes: lanes[pos] = {}
        
        cid = p['championId']
        chals = p.get('challenges', {})
        
        micro_score = 0
        macro_score = 0
        
        # --- ROLE-BASED FEATURE EXTRACTION ---
        if pos == "UTILITY":
            # Support Micro: Quest completion and Vision advantage
            if chals.get('fasterSupportQuestCompletion', 0): micro_score += 1
            if chals.get('visionScoreAdvantageLaneOpponent', 0): micro_score += 1
            # Support Macro: KP and Vision
            macro_score = chals.get('killParticipation', 0) * 100 + p.get('visionScore', 0)

        elif pos == "JUNGLE":
            # Jungle Micro: Early ganks and takedowns
            micro_score = chals.get('takedownsBefore15', 0) + chals.get('killsOnLanerEarlyJungleAsJungler', 0)
            # Jungle Macro: Objectives
            macro_score = p.get('damageDealtToObjectives', 0) + (chals.get('dragonTakedowns', 0) * 1000)

        elif pos == "MIDDLE":
            # Mid Micro: Classic lane dominance
            if chals.get('laningPhaseGoldExpAdvantage', 0): micro_score += 2
            micro_score += chals.get('soloKills', 0)
            # Mid Macro: Roaming and Objectives
            macro_score = (chals.get('takedownsBefore15', 0) * 500) + p.get('damageDealtToObjectives', 0)

        else: # TOP & BOTTOM
            # Laner Micro: Gold/CS/Kills
            if chals.get('laningPhaseGoldExpAdvantage', 0): micro_score += 2
            micro_score += chals.get('soloKills', 0)
            if chals.get('maxCsAdvantageOnLaneOpponent', 0) > 10: micro_score += 1
            # Laner Macro: Objective Damage
            macro_score = p.get('damageDealtToObjectives', 0)

        player_data = {"id": cid, "micro": micro_score, "macro": macro_score}

        if p['win']: lanes[pos]['winner'] = player_data
        else: lanes[pos]['loser'] = player_data

    # --- THE HEAD-TO-HEAD COMPARISON ---
    for pos, data in lanes.items():
        if 'winner' in data and 'loser' in data:
            w_data, l_data = data['winner'], data['loser']
            
            c_a_id = min(w_data['id'], l_data['id'])
            c_b_id = max(w_data['id'], l_data['id'])
            
            a_is_winner = (c_a_id == w_data['id'])
            a_data = w_data if a_is_winner else l_data
            b_data = l_data if a_is_winner else w_data
                
            # ML TRUTH LABELS
            a_won_game = a_is_winner
            a_won_micro = a_data['micro'] > b_data['micro']
            a_won_macro = a_data['macro'] > b_data['macro']

            try:
                supabase.rpc("update_matchup_holistic", {
                    "c_a": c_a_id, "c_b": c_b_id, "pos": pos, 
                    "t": tier, "dur": dur_bucket,
                    "a_won_game": a_won_game, 
                    "a_won_micro": a_won_micro, 
                    "a_won_macro": a_won_macro
                }).execute()
            except Exception as e:
                print(f"Matchup Error for {pos}: {e}")
